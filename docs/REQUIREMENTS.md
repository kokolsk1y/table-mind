# Requirements — TableMind

## V1 — Must Ship (неделя 1, до начала рассылки)

| ID | Requirement | Notes |
|----|-------------|-------|
| MENU-01 | Каталог меню из catalog.json с фото, ценой, описанием, тегами | Статический JSON, ~30-150 позиций |
| MENU-02 | Фильтры по категориям (горячее, салаты, напитки...) | Табы или чипы сверху |
| MENU-03 | Фильтры по тегам (веган, острое, без глютена) | Комбинируются с категориями |
| MENU-04 | Поиск по названию и описанию блюда | MiniSearch, fuzzy, prefix |
| AI-01 | Инлайн AI: нажатие на карточку → bottom sheet с описанием от AI | Стриминг, один запрос на блюдо |
| AI-02 | AI-чат: полноценный диалог с Официантом | Стриминг, selectItemsForAI(), history |
| AI-03 | Жёсткая привязка AI к данным меню | Всё меню в system prompt, запрет выдумывать |
| AI-04 | Асинхронный верификатор ответов | Fire-and-forget, ⚠️ при несоответствии |
| AI-05 | Дисклеймер по аллергенам в UI | Постоянный футер, не popup |
| AI-06 | catalog.json: поле allergens (массив) обязательно | Пустой = «неизвестно», не «нет» |
| ORDER-01 | Корзина «мой выбор» (локально) | Добавить/убрать блюда, sessionStorage |
| ORDER-02 | Кнопка «позвать официанта» → Telegram | Стол №, список блюд, через /api/notify |
| ORDER-03 | Retry-кнопка если официант не подошёл | Повторная отправка Telegram |
| QR-01 | QR-коды с параметром стола (?table=N) | Генерация набора для ресторана |
| QR-02 | Передача номера стола через sessionStorage | Не localStorage (стол = сессия) |
| PWA-01 | PWA: установка, офлайн-меню, service worker | Меню офлайн, AI только онлайн |
| PWA-02 | Загрузка <2 сек при повторном визите | Service worker cache |
| PWA-03 | iOS: инструкция «Добавить на Home Screen» | Overlay при определении iOS |
| DEPLOY-01 | Фронтенд на GitHub Pages | adapter-static, .nojekyll, appDir: 'app' |
| DEPLOY-02 | API на Vercel Functions (Node.js runtime) | НЕ Edge. Регион fra1 |
| DEPLOY-03 | CORS настроен (GitHub Pages → Vercel) | Access-Control-Allow-Origin |
| UI-01 | Тёмная ресторанная тема | DaisyUI v5, кастомная палитра |
| UI-02 | Мобильный-first, адаптивный | Основной юзкейс — телефон |
| UI-03 | Демо-контент: вымышленный ресторан | Переиспользуемый для рассылки |
| UI-04 | Режим демо без ?table | Кнопка «позвать официанта» заблокирована с пояснением |

## Out of Scope

- Интеграция с POS-системами (Poster, iiko, Quick Resto)
- Онлайн-оплата / приём заказов на кухню
- Корзина с интеграцией в приложение ресторана
- Раздельные меню (зал/бар/терраса)
- Кастомный дизайн под каждый бренд
- Автоматический парсинг меню с сайтов
- n8n workflow автоматизация
- Админ-панель для ресторатора (меню обновляется через JSON)
