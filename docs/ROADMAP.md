## Фаза 1: Фундамент

**Цель:** Проект задеплоен на GitHub Pages + Vercel, PWA устанавливается, QR-параметр стола считывается из URL.

**Зависимости:** Нет

**Требования:**
- DEPLOY-01: Фронтенд на GitHub Pages (adapter-static, .nojekyll, appDir: 'app')
- DEPLOY-02: API на Vercel Functions, Node.js runtime, регион fra1
- DEPLOY-03: CORS настроен (GitHub Pages → Vercel)
- UI-01: Тёмная ресторанная тема (DaisyUI v5, кастомная палитра)
- UI-02: Мобильный-first, адаптивный layout
- PWA-01: PWA: установка, офлайн-меню, service worker
- PWA-02: Загрузка <2 сек при повторном визите (service worker cache)
- QR-01: Генерация набора QR-кодов с параметром ?table=N
- QR-02: Передача номера стола через sessionStorage

**Критерии готовности:**
1. `npm run build` завершается без ошибок, `npm run preview` показывает страницу
2. Lighthouse PWA score ≥ 90 на мобильном (команда `npx lighthouse <url> --only-categories=pwa`)
3. GET `https://<vercel-domain>/api/chat` возвращает HTTP 405 или 200 (CORS header присутствует)
4. Открытие `/?table=5` → в sessionStorage записывается `tableNumber = "5"`
5. (Человек) PWA устанавливается на Android Chrome и доступна офлайн

**Ключевые артефакты:**
- `svelte.config.js` — adapter-static, appDir, trailingSlash
- `vite.config.js` — @vite-pwa/sveltekit config
- `static/.nojekyll`
- `vercel-api/api/chat.js` — заглушка с CORS
- `src/routes/+layout.svelte` — считывает ?table, кладёт в sessionStorage
- `src/app.css` — Tailwind 4 + DaisyUI v5 кастомная тема

## Фаза 2: Каталог меню

**Цель:** Гость видит полное меню ресторана с фильтрами по категориям и тегам и мгновенным поиском.

**Зависимости:** Фаза 1

**Требования:**
- MENU-01: Каталог меню из catalog.json с фото, ценой, описанием, тегами
- MENU-02: Фильтры по категориям (горячее, салаты, напитки...)
- MENU-03: Фильтры по тегам (веган, острое, без глютена)
- MENU-04: Поиск по названию и описанию (MiniSearch, fuzzy, prefix)
- AI-06: Поле allergens (массив) в catalog.json обязательно (пустой = «неизвестно»)
- UI-03: Демо-контент — вымышленный ресторан, 30+ позиций, все поля заполнены
- UI-04: Режим демо без ?table — кнопка «позвать официанта» заблокирована с пояснением

**Критерии готовности:**
1. `npx playwright test menu.spec.ts` (или аналог): фильтр «веган» показывает только вегетарианские блюда
2. Поиск по «том» находит «Том-ям» при 1 опечатке (fuzzy)
3. Открытие без `?table` — кнопка «позвать официанта» отображается disabled с текстом-пояснением
4. (Человек) catalog.json содержит ≥ 30 позиций с заполненными полями включая allergens

**Ключевые артефакты:**
- `static/catalog.json` — демо-меню вымышленного ресторана
- `src/lib/search/engine.js` — MiniSearch инициализация
- `src/lib/search/filters.js` — фильтры по категориям/тегам
- `src/lib/data/catalog.js` — loadCatalog() singleton
- `src/routes/+page.svelte` — главная страница с каталогом
- `src/lib/components/MenuCard.svelte`

## Фаза 3: AI-официант

**Цель:** Нажатие на карточку блюда открывает инлайн-описание от AI со стримингом, чат с Официантом работает, верификатор асинхронно помечает неточности.

**Зависимости:** Фазы 1, 2

**Требования:**
- AI-01: Инлайн AI — нажатие на карточку → bottom sheet с описанием, стриминг
- AI-02: AI-чат — полноценный диалог с Официантом, стриминг, история, selectItemsForAI()
- AI-03: Жёсткая привязка AI к данным меню — system prompt с правилом не выдумывать
- AI-04: Асинхронный верификатор — fire-and-forget, значок ⚠️ при несоответствии
- AI-05: Дисклеймер по аллергенам — постоянный футер, не popup
- PWA-03: iOS — overlay «Добавить на Home Screen» при определении iOS Safari

**Критерии готовности:**
1. `curl -X POST https://<vercel>/api/chat -d '{"agent":"waiter","message":"расскажи про борщ","catalog":[...]}' -H 'Content-Type: application/json'` возвращает стриминговый ответ без упоминания несуществующих блюд
2. Ответ AI не содержит блюд, отсутствующих в переданном catalog (тест: catalog с 1 блюдом, вопрос о другом — AI говорит «нет в меню»)
3. (Человек) Bottom sheet открывается за <1 сек, закрывается свайпом вниз
4. (Человек) После получения ответа с выдуманным аллергеном — через ≤ 10 сек появляется значок ⚠️
5. (Человек) Постоянный дисклеймер аллергенов виден на всех экранах без скролла

**Ключевые артефакты:**
- `vercel-api/api/chat.js` — единый AI роутер с параметром agent
- `src/lib/ai/client.js` — streamChat()
- `src/lib/ai/prompt.js` — selectItemsForAI(), system prompt с правилом запрета выдумывать
- `src/lib/ai/agents.js` — промпты агентов (waiter, verifier)
- `src/lib/ai/parse.js` — extractDishes()
- `src/lib/components/InlineAI.svelte` — bottom sheet
- `src/lib/components/ChatMessage.svelte`
- `src/lib/components/VerificationBadge.svelte`
- `src/routes/chat/+page.svelte`

## Фаза 4: Корзина, заказ и финальное демо

**Цель:** Гость формирует список выбранных блюд и вызывает официанта через Telegram; MVP готов для рассылки потенциальным клиентам.

**Зависимости:** Фазы 1, 2, 3

**Требования:**
- ORDER-01: Корзина «мой выбор» локально (добавить/убрать блюда, sessionStorage)
- ORDER-02: Кнопка «позвать официанта» → Telegram (стол №, список блюд, через /api/notify)
- ORDER-03: Retry-кнопка если официант не подошёл (повторная отправка Telegram)

**Критерии готовности:**
1. `curl -X POST https://<vercel>/api/notify -d '{"table":3,"items":["Борщ","Стейк"]}' -H 'Content-Type: application/json'` → в Telegram-чат бота приходит сообщение «Стол 3: Борщ, Стейк»
2. /api/notify возвращает HTTP 200 при успехе, 500 при ошибке Telegram API
3. (Человек) Сценарий конец-в-конец: открыть ?table=7 → добавить 2 блюда → нажать «позвать официанта» → Telegram-сообщение получено
4. (Человек) Retry отправляет повторное сообщение с пометкой «(повторный вызов)»
5. (Человек) Демо-ссылка отправлена как минимум в 1 тестовое сообщение рассылки

**Ключевые артефакты:**
- `vercel-api/api/notify.js` — Telegram Bot API интеграция
- `src/lib/stores/cart.svelte.js` — состояние корзины
- `src/lib/components/CallWaiterButton.svelte`
- `src/lib/stores/session.svelte.js` — tableNumber, currentMode

## Фаза 5: Голос и память гостя

**Цель:** Гость может говорить с AI голосом на Android/Desktop, а при повторном визите видит персонализированное приветствие.

**Зависимости:** Фаза 4

**Требования:**
- VOICE-01: Голосовой ввод (Web Speech API) — tap-to-talk, single-shot, Android/Desktop Chrome
- VOICE-02: Feature detection + fallback к тексту на iOS (скрыть кнопку, не показывать ошибку)
- MEM-01: Память гостя (localStorage) — предпочтения, navigator.storage.persist()
- MEM-02: Приветствие при повторном визите — «В прошлый раз вам понравилось...»

**Критерии готовности:**
1. На iOS Safari кнопка микрофона не отображается и ошибка в консоли не появляется (проверка `window.SpeechRecognition === undefined`)
2. (Человек) На Android Chrome: нажатие на микрофон → запись → распознанный текст появляется в поле ввода
3. (Человек) Второй визит: открыть страницу, выбрать предпочтение, закрыть, открыть снова → появляется персонализированное приветствие

**Ключевые артефакты:**
- `src/lib/stores/guest.svelte.js` — предпочтения в localStorage
- `src/lib/components/VoiceInput.svelte` — tap-to-talk с feature detection

## Фаза 6: Квиз, мультиязычность, агент-менеджер, аналитика

**Цель:** Режим 3 (интерактивный тест) работает и собирает комплексы; AI отвечает на языке гостя; агент-менеджер отвечает на FAQ; вопросы логируются.

**Зависимости:** Фаза 5

**Требования:**
- QUIZ-01: Режим 3 — интерактивный тест (5 вопросов, один на экран, progress bar, визуальные карточки)
- QUIZ-02: Результат теста — 3 именованных комплекса (compromise effect, ценовой якорь)
- AGENT-01: Агент-менеджер (FAQ о заведении: время работы, бронь, Wi-Fi, парковка)
- LANG-01: Мультиязычные ответы AI — инструкция в промпте отвечать на языке гостя
- ANALYTICS-01: Логирование вопросов и просмотров (для ресторатора)

**Критерии готовности:**
1. `curl -X POST https://<vercel>/api/chat -d '{"agent":"waiter","message":"what is in the borscht?"}' -H 'Content-Type: application/json'` возвращает ответ на английском языке
2. `curl -X POST https://<vercel>/api/chat -d '{"agent":"manager","message":"когда вы закрываетесь?"}' -H 'Content-Type: application/json'` возвращает ответ с информацией из FAQ без упоминания меню
3. (Человек) Прохождение теста из 5 вопросов → отображаются 3 именованных комплекса с ценами
4. (Человек) В логах Vercel видны записи вопросов с полями: timestamp, агент, хэш сессии (без PII)

**Ключевые артефакты:**
- `src/routes/quiz/+page.svelte`
- `src/lib/ai/agents.js` — добавлен промпт агента-менеджера и инструкция мультиязычности
- `vercel-api/api/chat.js` — добавлено логирование

## Примечания к рискам

**Главный риск фазы 1:** Конфигурация SvelteKit adapter-static + GitHub Pages + Vercel — много движущихся частей. Решение: сначала задеплоить пустую страницу, проверить все пути, затем добавлять фичи.

**Главный риск фазы 3:** Стриминг из Vercel Functions на GitHub Pages — CORS + chunked transfer. Решение: протестировать стриминг в изоляции (curl) до интеграции в UI.

**Риск Vercel Hobby:** Hobby plan запрещает коммерческое использование. Демо-стенд на Hobby — допустимо. При первом платном клиенте — переход на Pro ($20/мес).

**Риск OpenRouter оплата:** Российские карты не проходят. Иметь USDT TRC-20 кошелёк заранее или пополнить до начала работы.
