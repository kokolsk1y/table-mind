# Онбординг ресторана — как быстро собрать каталог

Когда ресторан согласился на пилот, классическая проблема — шефу некогда писать описания каждого блюда. У нас есть CLI-скрипт, который генерирует описания, аллергены и теги через Claude Haiku. Шеф проверяет и правит, не пишет с нуля.

## За сколько онбординг был → стал

- **Было:** шеф пишет описания + заполняет аллергены + проставляет теги вручную. 60 блюд × 5 минут = 5 часов работы шефа
- **Стало:** шеф присылает голый список (название + категория + состав + цена) → скрипт генерирует всё остальное → шеф только проверяет. 60 блюд × 30 сек = 30 минут

## Шаг 1 — попросить у шефа минимум

Нужно JSON такого вида (или Excel/Google Sheets → конвертируем):

```json
[
  {
    "name": "Клопсы кёнигсбергские",
    "category": "Тёплые закуски",
    "ingredients": "телятина, каперсы, сливки, мука, яйца",
    "price": 590,
    "weight": "250 г"
  }
]
```

Поля обязательные: `name`, `category`, `ingredients`, `price`.
Необязательные: `weight`, `photo` (если есть фото — URL или путь).

Сохрани файл как `tmp/{restaurant-slug}-input.json`.

## Шаг 2 — запустить обогащение

```bash
OPENROUTER_API_KEY=sk-or-... npm run enrich -- tmp/teliani-input.json tmp/teliani-catalog.json
```

На выходе получишь полный `catalog.json` со всеми полями: описание, теги, аллергены, `spicy`, `vegetarian`, сгенерированный `id` (транслит от названия).

**Что скрипт делает:**
- Разбивает меню на пачки по 8 блюд (чтобы контекст Claude не переполнился)
- Каждую пачку отправляет в Claude Haiku 4.5
- Получает JSON-массив с обогащением
- Мерджит с исходниками (цены и состав не трогает, только добавляет)
- Пишет единый итоговый файл

**Стоимость:** ~$0.003 за блюдо на Haiku 4.5. Меню на 60 блюд = ~$0.18 (15 ₽). Время: ~1-2 минуты.

## Шаг 3 — дать шефу на проверку

**Обязательно!** AI может:
- Ошибиться в аллергенах (пропустить редкие — сельдерей, горчица)
- Написать общие фразы там, где нужна историческая специфика ("прусский рецепт" может стать "немецким")
- Проставить "хит" где попало (этот тег субъективен)

Открой `tmp/teliani-catalog.json` в редакторе, отправь шефу для проверки. Ему нужно пройти быстро:
- Аллергены ок? (**самое важное** — жизнь и смерть)
- Описание не врёт про ингредиенты?
- "Хит" проставлен на действительно ходовые позиции?

После правок — каталог готов к деплою в PWA.

## Шаг 4 — деплой

1. Положи финальный `catalog.json` в `static/catalog.json` (заменяет шаблонный «Янтарный берег»)
2. Обнови название ресторана в `src/lib/config.js` (или где сейчас хранится)
3. Обнови системный промпт в `vercel-api/api/chat.js` — замени «Янтарный берег» и адрес на актуальные
4. Сгенерируй QR-коды: `npm run generate-qr`
5. Деплой: `git push`, Vercel подхватит

## Частые проблемы

**OpenRouter отдал 429 (rate limit)** → подожди минуту, запусти снова, скрипт пишет результат в конце, не сохраняет частично

**Claude ответил не JSON** → бывает редко, скрипт кидает ошибку с raw-текстом. Запусти ещё раз, обычно со второго прохода ок

**Аллергены пропущены** → проверь что `ingredients` у тебя достаточно детальный. "мясо" недостаточно — нужно "говядина, мука, сливки"

**Описания похожи друг на друга** → повысь `temperature` в скрипте с 0.4 до 0.6 (более разнообразно, но менее контролируемо)

## Что скрипт НЕ делает (важно)

- **Не генерирует фото блюд.** Мы отказались от AI-фото — риск что гость ждёт один вид, а получает другой. Фото либо с сайта ресторана, либо шеф фотографирует быстро на iPhone.
- **Не переводит на другие языки.** Мультиязычие делает AI в рантайме на каждом запросе гостя — отдельный каталог для каждого языка не нужен.
- **Не проверяет цены и состав.** Скрипт доверяет входу. Если шеф прислал ошибку — она пойдёт в продакшн.
