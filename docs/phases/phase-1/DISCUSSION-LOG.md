# Discussion Log — Phase 1

**Date:** 2026-04-11

## Questions Raised and Answered

### Q: Как разделить SvelteKit фронт и Vercel Functions в одном репо? Три варианта: monorepo с подпапкой, два отдельных репо, всё на Vercel.
**A:** Пользователь выбрал monorepo (Вариант A). Уточнил опасение: «не будут ли пересекаться?». После объяснения, что Vercel изолирует подпапку через Root Directory и у каждой части свои зависимости и свой триггер деплоя — согласился.
**Decision:** Monorepo, SvelteKit в корне, Vercel Functions в `vercel-api/`, Vercel подключается с `Root Directory: vercel-api`.

### Q: Как генерировать QR-коды для столов? Build-time скрипт, runtime-админка или только инструкция?
**A:** Пользователь подтвердил, что нужен QR на каждый стол. На варианте C (просто инструкция) не остановился — значит продукт должен давать готовые QR.
**Decision:** Build-time скрипт `scripts/generate-qr.js` с параметрами `--tables N --base URL --output DIR`. Выход: PNG на каждый стол + PDF с сеткой для печати.

### Q: Есть ли готовое лого для PWA-иконок? Три варианта: есть лого, временный плейсхолдер сейчас, делаем красивое вручную.
**A:** Пользователь ответил: «я сгенерирую попозже».
**Decision:** Временный плейсхолдер на Фазе 1 (монограмма «TM» на тёмном фоне). Замена через `scripts/generate-icons.js` когда лого будет готово.

### Q: Какой визуальный стиль тёмной темы? Italian/Wine Bar, Modern Dark Premium, Japanese/Minimal, Warm/Bistro?
**A:** Пользователь принял мою рекомендацию — Modern Dark Premium как нейтральный вариант для любого сегмента.
**Decision:** Modern Dark Premium — почти чёрный фон, угольные карточки, янтарно-медный акцент, sans-serif (Inter или Space Grotesk), кастомная DaisyUI v5 тема через CSS-переменные для лёгкой перекраски под клиента.

## Questions Raised but Deferred

### Q: Настоящее лого TableMind?
**Reason deferred:** Пользователь сгенерирует отдельно. Временный плейсхолдер не блокирует деплой.
**Owner:** Пользователь, по готовности.

### Q: Custom domain для продакшена?
**Reason deferred:** На этапе демо достаточно `kokolsk1y.github.io/table-mind/`. Кастомный домен имеет смысл при первом платном клиенте.
**Owner:** Revisit при закрытии первого платного заказа.

### Q: Vercel Pro vs Hobby (коммерческое использование)?
**Reason deferred:** Hobby допустим для демо-стенда. При первом платном клиенте — миграция на Pro ($20/мес) или self-host на стороне клиента.
**Owner:** Revisit при первом платном клиенте.

### Q: Per-restaurant темизация (несколько клиентов на одном домене)?
**Reason deferred:** В v1 один деплой = один ресторан. Multi-tenant — не приоритет, появляется только если нужно обслуживать 3+ клиентов на одной инфраструктуре.
**Owner:** Revisit в V2 или по запросу.
